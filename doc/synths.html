Llia Synths

Llia has three general synth types:

  1. Regular "Instrument" synths
  2. Effect Synths
  3. Control Synths

The word "synth" may refer to any one of these types, hopefully context will make it clear which synth type is being described. Within each of these broad categories there are numerous specific synth types.

A regular synth is an instrument of some sort which may be played, such as an organ or synthesizer. Concrete examples are the MASA organ and Saw3 synthesizer. All regular synths have at least one audio output. They typically have at least one control input. Conceivably they might also have audio inputs or control outputs, but this would be rare and no current instrument have these ports.

An effect synth is used to process signals external to itself. The input signal may come from an instrument synth, another effect or from the sound card audio inputs. Effect synths will always have at least one audio input and one audio output. They typically have at least one control input, but not always. They might also have one or more control outputs.

Control synths will always have at least one control output. They may also have one or more control inputs. They will never have audio ports of any kind.

It is important to maintain a distinction between client side and server side synths. A server side Synth is an active object running on the SuperCollider server. We are rarely directly concerned with these as they are taken care of behind the scenes. The synths that do concern us are on the client. Technically these are instances of the python SynthProxy class. Unless specifically noted the term "synth" refers to these client-side proxies and not to the objects on the server.

All (client-side) synths share certain common features. These include:

  • A Program Bank
  • A "current" program
  • A keymode

A program bank is a collection of 128 Programs. Programs may be moved around within the bank or copy and pasted across banks of the same type. Banks as a whole may be saved to external files. A program contains values for all parameters of a synth. Each program also contains a Performance which is a collection of values related to how the synth responds to specific MIDI events.

A bank always has a current program which is selected either by MIDI program change events or by the user selecting it from a program list.

Each synth also has a keymode which is specified when the synth is created. The keymode controls how a synth is allocated in response to MIDI note events. Not all synths work with all possible keymodes. Effects and Control synths for example may only use the special efx keymode.
