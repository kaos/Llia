There are three general types of Llia synths:

  1. Regular "Instrument" synths
  2. Effect Synths
  3. Control Synths

The word "synth" may refer to any one of these types, hopefully context will make it clear which synth type is being described. Within each of these broad categories there are numerous specific synth types.

A regular synth is a playable instrument such as an organ or synthesizer.

An effect synth is used to process signals external to itself. The input signal may come from an instrument synth, another effect or from a hardware audio input. Effects will always have at least one audio input and one audio output.

Control synths will always have at least one control output. They may also have one or more control inputs.

It is important to maintain a distinction between client side and server side synths. A server side Synth is an active object running on the SuperCollider server. We are rarely directly concerned with these as they are taken care of behind the scenes. The synths that do concern us are on the client. Technically these are instances of the python SynthProxy class. Unless specifically noted the term "synth" refers to these client-side proxies and not to the objects on the server.

All (client-side) synths share certain common features. These include:

  • Program Bank
    Each type of synth has a specific Program type which contains values for all of it's parameters.
    Each Program contains a Performance which determines how the synth responds to specific MIDI events.
    Programs are collected together into banks. Every instance of an instrument has its own bank of 128 Programs.
  • current program. A bank always has a current program. Initially this is program 0.
  • A keymode which determines how the synth responds to MIDI key events.
