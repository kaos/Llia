Keymodes

Each active synth has an associated keymode which manages how the synth responds to MIDI key events. Not every synth type is compatible with every keymode.

Currently there are four keymodes, each with it's own set of pros and cons. Additional keymodes may be added later.

Poly1 is the default mode for most instrumental synths. Server side synths are created on the fly as new keys are pressed. After the key has been released, and all envelope segments have terminated, the synth object is freed.

Pros:

  • Simple
  • Up to 128 note polyphony
  • May be used with any effect regardless of synth creation order

Cons:

  • May not be fast enough for complex synths.
  • Effects which require voice history, such as portamento, do not work
  • Complex synths with long release times may have a high resource drain

PolyRotate is an alternate polyphonic mode which creates a fixed number of server side synths ahead of time. The voices are allocated in strict rotation in response to MIDI note events. Non playing voices remain in memory.

Pros:

  • Simple, only slightly more complex then Poly1
  • Fast, no need to create synth objects on the fly
  • Fixed resource load
  • Portamento supported (in a fashion)

Cons:

  • Fixed polyphony, voices are stolen whenever key down count exceeds voice count

Mono1 is a monophonic keymode with last note priority. A single persistent server side synth object is created.

Pros:

  • Fast
  • Fixed resource load
  • Portamento supported

Cons:

  • Monophonic

Efx is a specialized keymode for effects and control synths. As with mono mode a single persistent server side synth object is created. Efx mode passes the last keynumber, note frequency, and velocity to the effect. As long as any key is down the gate signal is high, if all keys are up the gate signal goes low.
